    What is ClinkWorks?

    We want to deliver a full fledged playground for any hobbiest at any level to learn complex java skills and knowledge without the challenge of software hindering your ability to learn a new software development concept

    We really want to deliver developers creating rest API's using the ClinkWorks-Platform, easy means to create their own applications with out of the box support for valuable development techniques and education about API design.

    The idea is that it is possible to create a simple system with minimum viable product design with modular service based architecture at the for front of our api choices. Taking this approach with strict test driven development with a focus on a service based architecture.

    We have decided that it was time to take our hobbiest nature to the next level. We want to provide tooling and know how to developers of any skill level seeking to build their own products

    What are we creating

    We are attempting to create a data broker called ClinkWorks-Platform that is as close to Java SE compliant as we can make it without adding any strict learning curves in any of the given supported technologies.

    One of the big things I felt as a new developer in the enterprise world is just how many different xml configuration files you were required to learn at a very quick pace. Xml files for conifguring databases, xml files for identifying constructors. xml files for this that and the other.

    Modern application development tooling is also so difficult to learn since you are never exposed to it in the achedemic world. So we decided to host a full fledged application stack including a CI server, multiple enviroment configurations for the build, and all that fun stuff.

    For those of you interested, here is an overview of the overall architecture for the platform:

    Server

    • RESTful services are provided through the JAX-RS API through the reference implementation Jersey
    • Object to relational data mapping provided through the Java Persistence API via the reference implementation Apache OpenJPA
    • Object serialization/deserialization provided through the JAXB API via the Jackson Object Serialization API
    • Configuration and dependency management provided through JSR-299 via Google Guice
    • Collections management provided by Google Guava
    • Build Management - Maven
    • Continuous Build Server - Jenkins
    • Project management and task tracking Kanbanize (free and awesome btw - lacking features but has enough for a 2 man team)
    • Project hosting by Amazon EC2 Cloud
    • Development Server - Jetty
    • Development Database - H2
    • Production Server - Tomcat
    • Production Database - PostgreSQL
    • TDD software - JUnit
    • Mock framework - JMockIt

    Client

    • Production Server - Apache Server
    • Test Server - Karma
    • MVC provided by Angular
    • Object serialization/deserialization provided through Angular.$resource
    • Data binding provided by Angular
    • REST API management provided by Angular.$resource
    • Collections management provided by LoDash via a ClinkWorks Service
    • SERVER/UI Data Management done via a ClinkWorks domain model
    • Build Management - Grunt
    • TDD software - Jasmine
    • Dependency Management - NPM
    • Mock Framework - Angular.$mock
